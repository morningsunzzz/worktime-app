# 考勤打卡应用 - 产品需求文档

## 1. 产品概述

### 1.1 产品定位
一款轻量级个人考勤打卡 PWA（Progressive Web App），安装在手机上使用，帮助个人记录每日上下班时间、自动计算加班时长、提供月度统计。

### 1.2 目标用户
- 需要记录个人考勤的上班族
- 弹性工作制或需要精确记录加班时间的员工
- 不依赖公司打卡系统、希望有独立考勤记录的人

### 1.3 核心价值
- **简单**: 打开即打卡，无多余操作
- **离线可用**: PWA 方案，无网络也能打卡，联网后自动同步
- **数据自有**: 数据存储在本地 IndexedDB，隐私安全
- **无需安装**: 浏览器访问即可使用，支持添加到手机主屏幕

## 2. 功能需求

### 2.1 核心功能

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0 | 上班打卡 | 点击按钮记录上班时间，按钮状态变为"已上班" |
| P0 | 下班打卡 | 点击按钮记录下班时间，自动计算当日工时 |
| P0 | 当日状态展示 | 显示今日打卡时间、已工作时长、预计下班时间 |
| P1 | 加班计算 | 基于标准工时(默认8h)，自动计算加班时长 |
| P1 | 历史记录 | 按月份查看打卡记录列表 |
| P1 | 月度统计 | 月度出勤天数、总工时、加班总时长 |
| P2 | 编辑记录 | 修改某条打卡时间（补卡） |
| P2 | 导出数据 | 导出为 CSV/Excel |
| P2 | 备注 | 打卡时可备注（如：请假、出差、外勤） |
| P3 | 多班次 | 支持午休分段打卡 |
| P3 | 提醒通知 | 定时提醒打卡 |

### 2.2 数据模型

```
WorkRecord {
  id: string (uuid)
  date: string ("2026-05-15")
  clockIn: datetime ("2026-05-15T09:05:00")
  clockOut: datetime | null
  totalHours: number | null (工时，精确到 0.5h)
  overtimeHours: number | null (加班时长)
  note: string | null
  createdAt: datetime
  updatedAt: datetime
}
```

### 2.3 设置项

| 设置 | 默认值 | 说明 |
|------|--------|------|
| 标准日工时 | 8 小时 | 超出部分算加班 |
| 午休时长 | 0 分钟 | 是否从工时中扣除午休 |
| 上班提醒 | 关闭 | 定时提醒上班打卡 |
| 下班提醒 | 关闭 | 定时提醒下班打卡 |

## 3. 交互设计要点

### 3.1 首页（打卡页）
- 居中大按钮：蓝色"上班打卡" / 红色"下班打卡"
- 按钮上方显示当前时间和日期
- 按钮下方显示今日状态：上班时间、已工作X小时X分
- 底部导航：打卡 / 记录 / 统计 / 设置

### 3.2 记录页
- 按月分组展示打卡记录
- 每条记录显示：日期、上班时间、下班时间、工时、加班标记
- 支持左右滑动切换月份
- 长按可编辑某条记录

### 3.3 统计页
- 本月概览卡片：出勤天数、总工时、加班总时长
- 简单柱状图：近30天每日工时趋势

### 3.4 设置页
- 标准工时设置
- 午休扣除设置
- 数据导出
- 关于

---

## 4. 非功能需求

- **离线可用**: 所有功能在离线状态下正常使用
- **响应速度**: 打卡操作 < 200ms 响应
- **数据安全**: 纯本地存储，不上传任何数据
- **跨平台**: iOS Safari + Android Chrome 均可安装为 PWA
- **体积**: 首屏加载 < 500KB
